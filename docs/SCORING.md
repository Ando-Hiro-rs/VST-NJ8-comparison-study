# スコアリング理論

このドキュメントでは NeoCELP と NeoVST-NJ8 のスコアリングアルゴリズムを説明します。

## NeoCELP — 反応時間データのクレンジング

NeoCELP の反応時間データには、横川 (2006) の標準手続きに準拠した4段階クレンジングを適用します。

### ステップ1: 不正解の除外

正答した試行のみを以降の分析対象にします。

### ステップ2: 100ms以下のフライング応答を除外

人間の生理学的反応限界 (約100〜150ms) を下回る応答は、刺激処理を経ていない「フライング」とみなし除外します。

### ステップ3: 平均±3SDの範囲外を除外

残ったデータから平均と標準偏差 (SD) を算出し、`平均 ± 3SD` の範囲外を統計的外れ値として除外します。

### ステップ4: ACRRT と CV の算出

最終的に残った有効データから以下を算出します。

```
ACRRT = 正答試行の反応時間の平均 (ms)
CV    = SD / ACRRT × 100 (%)
```

- ACRRT: 値が小さいほど処理が速い
- CV: 値が小さいほど処理が安定している (個人内変動が小さい)

## NeoVST-NJ8 — 3PL IRT スコアリング

### 3パラメータロジスティックモデル

各項目への正答確率は以下で表されます。

```
P(X_j = 1 | θ) = c_j + (1 - c_j) / (1 + exp(-1.7 · a_j · (θ - b_j)))
```

- `a_j`: 識別力 (項目が能力差をどれだけ識別できるか)
- `b_j`: 困難度 (正答確率が中央になる能力値)
- `c_j`: 当て推量パラメータ (能力が極めて低い受験者の正答確率)
- `θ`: 受験者の潜在能力 (語彙力)
- `1.7`: ロジスティック分布を正規分布に近似する定数

### θ の推定 (最尤推定)

受験者が応答した試行データから、対数尤度を最大化する θ を探索します。

```
log L(θ) = Σ_j [ X_j · log P_j(θ) + (1 - X_j) · log (1 - P_j(θ)) ]
```

実装では θ を -3.0 から +3.0 まで 0.1 刻みで動かし、対数尤度が最大となる θ を採用します (グリッドサーチ MLE)。JACET 公式 Excel シートと互換性を保つため、最後に `+0.2` の補正を加えます。

### 標準誤差 (SE) の算出

Fisher 情報量から SE を計算します。

```
I(θ) = Σ_j [ (1.7 · a_j · (P_j - c_j) / (1 - c_j))^2  · (1 - P_j) / P_j ]
SE(θ) = 1 / √I(θ)
```

SE は θ の推定精度を表します。SE が小さいほど推定が安定しています。

### 語彙サイズの推定

各レベル (1〜8) の項目の平均パラメータを使って、そのレベル (1000語) のうち何語を知っているかを推定します。

```
レベル n の推定語彙数 = ROUND(P(θ; ā_n, b̄_n, c̄_n) × 1000)
合計語彙サイズ = Σ レベル1〜8の推定語彙数
```

ここで `ā_n, b̄_n, c̄_n` はレベル n に属する 20 項目の平均パラメータです。

## VST-NJ8 シャッフル整合性

選択肢の表示順をシャッフルする際、データの整合性を保証するため以下の3原則を実装しています。

### 原則1: シャッフル後の配列のみを永続化

シャッフル前の元配列は完全に破棄し、画面に表示された順序の `shuffled_choices` オブジェクトだけを保持します。

### 原則2: 応答記録に「選んだ単語そのもの」を含める

インデックスだけでなく、文字列としての単語そのものを記録します。これにより、CSV を後から見ても何を選んだか必ず特定できます。

### 原則3: 正誤判定は文字列一致

`response_word === correct_word` で正誤判定します。インデックス比較は使いません。

### バリデーション

サーバー (または validateVstIntegrity 関数) で、受信した試行データに以下をチェックします。

- `displayed_options[response_position]` が `response_word` と一致するか
- `response_word` が `displayed_options` に含まれるか
- `correct_word` が `displayed_options` に含まれるか
- `is_correct` が `response_word === correct_word` と一致するか

1つでも不一致があれば、データに矛盾があるためエラーとして検出します。
